System Overview

The RAG (Retrieval-Augmented Generation) System is a sophisticated document intelligence platform that combines semantic search with large language model capabilities. It allows users to upload documents, index their content, and ask natural language questions to retrieve relevant information.

Key Features

  • Multi-format Support: Process PDF and text files
  • Semantic Search: Vector-based document retrieval using sentence transformers
  • LLM Integration: Intelligent answer generation using Ollama models
  • Scalable Architecture: Built on FastAPI with ChromaDB vector storage
  • Privacy Protection: Built-in PII anonymization and data filtering
  • Real-time Processing: Streamlined ingestion and query pipeline

Architecture Components

FastAPI Backend

High-performance Python web framework handling API requests and business logic

ChromaDB

Vector database for efficient semantic search and document storage

Sentence Transformers

all-MiniLM-L6-v2 model for generating document embeddings

Ollama LLM

Local large language model for intelligent response generation

Authentication

The RAG System uses API key authentication for secure access. Include your API key in the request headers when making API calls. 

# Set your API key
export OLLAMA_API_KEY="your-api-key-here"

# Include in API requests
curl -H "Authorization: Bearer $OLLAMA_API_KEY" \
     -H "Content-Type: application/json" \
     http://localhost:8000/query

API Endpoints

Document Ingestion

Upload and process documents into the vector database for semantic search and retrieval.

POST /ingest multipart/form-data

Upload PDF or text files for processing and indexing

Request Parameters

Parameter Type Required Description
files File[] Yes PDF or text files to process

Response Format

{
  "inserted_chunks": 45,
  "total_tokens_approx": 12500,
  "vector_dim": 384,
  "chunk_ids": [
    "a1b2c3d4e5f6...",
    "g7h8i9j0k1l2...",
    "m3n4o5p6q7r8..."
  ]
}

Query System

Submit natural language questions to retrieve relevant information from indexed documents.

POST /query application/json

Submit queries and receive AI-generated answers with source attribution

Request Body

{
  "question": "What are the main findings in the research paper?",
  "k": 3,
  "score_threshold": 0.6,
  "filter": {
    "source": "research_paper.pdf",
    "min_page": 1,
    "max_page": 10
  }
}

Response Format

{
  "answer": "The main findings indicate significant improvements in accuracy...",
  "sources": [
    {
      "chunk_id": "a1b2c3d4e5f6...",
      "source": "research_paper.pdf",
      "page_number": 5,
      "date_added": "2024-01-15T10:30:00Z",
      "similarity": 0.8923,
      "snippet": "Our experiments demonstrate a 23% improvement..."
    }
  ]
}

Code Examples

Python Client

import requests
import json

class RAGClient:
    def __init__(self, base_url, api_key):
        self.base_url = base_url
        self.headers = {
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json'
        }
    
    def ingest_documents(self, file_paths):
        """Upload and process documents"""
        files = [('files', open(fp, 'rb')) for fp in file_paths]
        response = requests.post(
            f'{self.base_url}/ingest',
            headers={'Authorization': self.headers['Authorization']},
            files=files
        )
        return response.json()
    
    def query(self, question, k=3, threshold=0.6):
        """Submit a query and get AI response"""
        data = {
            'question': question,
            'k': k,
            'score_threshold': threshold
        }
        response = requests.post(
            f'{self.base_url}/query',
            headers=self.headers,
            json=data
        )
        return response.json()

# Usage example
client = RAGClient('http://localhost:8000', 'your-api-key')

# Upload documents
result = client.ingest_documents(['document1.pdf', 'document2.txt'])
print(f"Processed {result['inserted_chunks']} chunks")

# Query the system
response = client.query("What are the key findings?")
print(f"Answer: {response['answer']}")
print(f"Sources: {len(response['sources'])} documents")

JavaScript Client

class RAGClient {
    constructor(baseUrl, apiKey) {
        this.baseUrl = baseUrl;
        this.headers = {
            'Authorization': `Bearer ${apiKey}`,
            'Content-Type': 'application/json'
        };
    }

    async ingestDocuments(files) {
        const formData = new FormData();
        files.forEach(file => formData.append('files', file));
        
        const response = await fetch(`${this.baseUrl}/ingest`, {
            method: 'POST',
            headers: {
                'Authorization': this.headers['Authorization']
            },
            body: formData
        });
        
        if (!response.ok) {
            throw new Error(`Upload failed: ${response.statusText}`);
        }
        
        return await response.json();
    }

    async query(question, k = 3, threshold = 0.6) {
        const data = {
            question: question,
            k: k,
            score_threshold: threshold
        };
        
        const response = await fetch(`${this.baseUrl}/query`, {
            method: 'POST',
            headers: this.headers,
            body: JSON.stringify(data)
        });
        
        if (!response.ok) {
            throw new Error(`Query failed: ${response.statusText}`);
        }
        
        return await response.json();
    }
}

// Usage example
const client = new RAGClient('http://localhost:8000', 'your-api-key');

// Upload documents
const fileInput = document.getElementById('file-input');
const files = Array.from(fileInput.files);

try {
    const result = await client.ingestDocuments(files);
    console.log(`Processed ${result.inserted_chunks} chunks`);
    
    const response = await client.query('What are the key findings?');
    console.log('Answer:', response.answer);
    console.log('Sources:', response.sources.length);
} catch (error) {
    console.error('Error:', error.message);
}

Data Models

QueryRequest

Field Type Description
question string The natural language question to ask
k integer Number of relevant chunks to retrieve (default: 3)
score_threshold float Minimum similarity score for results (default: 0.6)
filter object Optional metadata filters for source documents

QueryResponse

Field Type Description
answer string AI-generated answer to the question
sources array List of source documents and chunks used

SourceItem

Field Type Description
chunk_id string Unique identifier for the text chunk
source string Original document filename
page_number integer Page number in the source document
similarity float Semantic similarity score (0-1)
snippet string Text content of the retrieved chunk

Error Handling

The API uses standard HTTP status codes and returns detailed error information in JSON format.

400 Bad Request

Invalid request parameters or missing required fields

401 Unauthorized

Missing or invalid API key authentication

500 Internal Server Error

Server-side error during processing

Error Response Format

{
  "detail": "Invalid request format",
  "status": 400,
  "error": "ValidationError"
}
